  ## Why Multiple Submissions?

  Most problems have 3-9+ submissions. This isn't retrying until it passes — each submission represents a **different
  approach or optimization**:

  - **Coin Change**: BFS-based exploration, bottom-up DP, and optimized tabulation
  - **Climbing Stairs**: Naive recursion, memoization, then iterative DP
  - **Binary Tree Maximum Path Sum**: Multiple DFS strategies with varying edge-case handling

  This reflects how I actually study — understanding the trade-offs between approaches, not just finding one that works.
